# Aurora 아키텍처

## 전체 흐름

```
┌──────────────────────────────────────────────────┐
│  Aurora.exe (Pywebview 윈도우)                  │
│  ┌────────────────────────────────────────────┐  │
│  │ Frontend: HTML / Tailwind / Vanilla JS     │  │
│  │ - 게임 옵션창 스타일 GUI                    │  │
│  │ - 카테고리 탭 + 슬라이더 + 토글              │  │
│  └─────────────────┬──────────────────────────┘  │
│                    ↓ HTTP / JS Bridge            │
│  ┌────────────────────────────────────────────┐  │
│  │ Backend: Python 3.11 + FastAPI             │  │
│  │  ┌───────────────────────────────────────┐ │  │
│  │  │ core/      신호 산출 · 리스크 계산      │ │  │
│  │  │ exchange/  거래소 호출 · 데이터 캐시    │ │  │
│  │  │ backtest/  과거 데이터 시뮬             │ │  │
│  │  │ interfaces/ API + Telegram + Webview   │ │  │
│  │  └───────────────────────────────────────┘ │  │
│  └─────────────────┬──────────────────────────┘  │
└────────────────────┼──────────────────────────────┘
                     │
                     ▼
            ┌────────────────────┐
            │ Telegram Bot       │   같은 백엔드 호출
            │ - 명령 + 알림       │
            └────────────────────┘
                     │
                     ▼
       ┌─────────────────────────────┐
       │ 거래소 API (ccxt 통합)        │
       │ Bybit / OKX / Binance        │
       └─────────────────────────────┘
```

## 데이터 흐름 (1 거래 사이클)

1. **시세 수집** — `exchange.data.MultiTfCache` 가 1H~1W OHLCV를 거래소에서 받아 캐시
2. **지표 계산** — `core.indicators` 가 EMA/RSI/Div/BB/MA/Harmonic/Ichimoku 계산
3. **신호 검출** — `core.strategy` 가 진입 조건 평가
4. **신호 합성** — `core.signal` 이 OR 조합으로 최종 결정
5. **리스크 계획** — `core.risk` 가 SL/TP/사이즈/트레일링 산출
6. **주문 실행** — `exchange.execution` 이 진입 + TP 4개 + SL 등록
7. **포지션 추적** — 트레일링 갱신 (`core.risk.update_trailing_sl`)
8. **청산 처리** — TP/SL/반대신호 도달 시
9. **알림** — `interfaces.telegram` 이 진입/청산 메시지 전송

## 모듈 의존성

```
interfaces ──→ core
           ──→ exchange
           ──→ backtest

backtest   ──→ core
           ──→ (exchange.data만 — paper)

exchange   ──→ (외부 ccxt만)

core       ──→ (외부 의존 0 — 순수 함수)
```

**중요**: core는 어떤 모듈에도 의존하지 않음 → 단위 테스트 쉽고, 백테스트와 라이브 모드에서 동일한 로직 재사용 가능.

## 실행 모드 (`run_mode`)

| 모드 | 의미 |
|---|---|
| `paper` | 신호만 발생, 주문 안 보냄 (개발 기본값) |
| `demo` | 거래소 testnet에서 실제 주문 |
| `live` | 실거래 (소액부터) |

`.env`의 `RUN_MODE` 값으로 전환.
